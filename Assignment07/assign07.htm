CSIS 352 Spring 2020 Assignment 7

Introduction

In this assignment, you will implement data structures and code to support a solution to the Josephus problem.

Purpose and Objectives

The purpose of the code is to implement code to facilitate a solution to the Josephus problem. The purpose of having you do this as an assignment is to ensure that you can correctly do the following:
  • implement a templated circuluar queue class
  • work with linked lists
  • sort
  • think about how a larger problem and the smaller pieces of it relate

Stakeholders

If you were to be in the situation that Josephus hypothetically was, this could be a matter of life and death for you. Have a good implementation ready to prevent your death in the future. (Note: this section is intended to have humor. Read the background for more information, and contact the instructor if you have any questions about what is humorous versus not humorous in this section.)

Where

This will be tested on smaug, and must execute there.

Tasks

The following files are provided for you:
  • makefile
  • person.h
  • circularqueue.h
  • main.cpp
  • testinput.txt

The following files are the ones that you are to write/improve:

  • circularqueue.h
  • main.cpp
  • README
  • testinput.txt (editing this one is optional)
Do not write any other files.

For background, read about the Josephus problem. We will be implementing a solution that involves the use of a circular queue that supports multiple variants of the problem. If you have any questions, feel free to ask the instructor in class.

Make no changes to person.h

Make no changes to makefile

If there are bizarre situations that you realize certain methods may find themselves in, then have them throw appropriate exceptions (see https://github.com/andrew-chen/csis352/tree/master/examples/exception for examples of appropriate exceptions and how to throw them).

You are writing most of the code in circularqueue.h and main.cpp as described in the comments in those files. It is your responsibility to figure out how to test methods in found in circularqueue.h that are not called in main.cpp.

Standards and Testing

Your code will be tested in the directory you submit with the files you submit. Additionally, your circularqueue.cpp file will be tested separately in a different environment, for more thorough testing. If you make any assumptions about input, be sure to check those assumptions and throw exceptions as appropriate. Multiple bizarre input possibilities will be provided during the independent testing of circularqueue.h

Known Issues and Risks

  • 0 or negative values for the number of people or other arguments
  • Number of names, or length of an individual name, a ridiculously large size
  • Leaving debug code in when it shouldn't be there (nothing should be output to cout or cerr unless directions explicitly say so)
  • Issues from the previous assignment
  • Ensuring that the next and previous pointers in the nodes are updated correctly
